# Sparker 对话式功能测试手册

> 场景设定：资深直播运营专家（8年经验）将全部 know-how 传授给 OpenClaw Agent

## 使用方法

依次将下列 Query 发送给你的 OpenClaw Agent（通过飞书、命令行或任何已连接的 channel），观察 Agent 的响应行为是否符合预期。

---

## 测试 1: 直接教学 — 验证 Kindle 采火

**Query:**

```
我是做直播运营的，有8年经验。我教你几个关键经验：直播标题千万不要超过15个字，太长用户根本不看。而且标题里一定要有数字，比如"3招搞定"比"教你几个方法"点击率高40%。
```

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|
| 记录为 RawSpark | ✓ | source=human_teaching, domain=直播策划.标题 |
| 主动追问边界 | ✓ | 例如"什么品类下这个规律最明显？有没有例外？" |
| 复述确认 | ✓ | 例如"我理解了：标题控制在15字以内 + 必须含数字。对吗？" |
| confidence 初始值 | ✓ | 0.70 (human_teaching 0.50 + confirmed 0.20) |

## 测试 2: 连续教学 — 多知识点结构化

**Query:**

```
再告诉你直播间留人的三板斧：
1. 开场前30秒直接说今晚最大的福利是什么
2. 每隔15分钟做一次"截屏抽奖"互动
3. 快下播前放终极福利，把人留到最后
```

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|
| 分别记录 3 条 RawSpark | ✓ | 每个知识点独立记录 |
| 追问场景适用性 | ✓ | "这个策略适合所有品类吗？知识付费和电商的留人策略有区别吗？" |
| 关联为 spark chain | 可能 | 识别为一组相关策略 |

## 测试 3: 用户纠正 — 验证反馈采集

**先出一个任务：**

```
帮我写一个今晚8点美妆直播的标题
```

**Agent 给出标题后，发送纠正：**

```
不行，你写的标题里有句号，直播标题绝对不能有句号，句号会让人觉得事情结束了，没有继续看的动力。另外"推荐"这个词太弱了，要用"必入""断货前"这种紧迫感的词
```

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|
| 记录为反馈 spark | ✓ | source=human_feedback |
| 立即修改标题 | ✓ | 应用刚学到的规则重新输出 |
| 与已有标题知识交叉验证 | 可能 | 和测试1的知识关联 |

---

## 测试 4: 多选偏好 — 验证 RL 引擎

**Query:**

```
帮我策划一个双十一大促的直播脚本框架
```

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|
| 给出 2-3 个不同方案 | ✓ | 如：方案A 低价秒杀 / 方案B 品牌故事 / 方案C 达人测评 |
| 等待用户选择 | ✓ | 不直接执行某一个方案 |
| 记录选择偏好 | ✓ | 用户选 B 后记录为 human_choice spark |

**选择一个方案后说：**

```
我选B，品牌故事路线更符合我们的定位
```

## 测试 5: 数据反馈 — 验证实践记录

**Query:**

```
昨天的直播数据出来了，用你之前帮我写的那个带数字的标题，点击率从3.2%涨到了5.1%。但那个暖色调封面的效果一般，只涨了0.3%
```

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|
| 数字标题策略 positive practice | ✓ | confidence 提升 |
| 暖色调封面 partial practice | ✓ | confidence 微调 |
| 追问细化 | 可能 | "封面是晚8点场还是白天场？如果是白天场暖色调可能不合适" |

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|
| 记录为教学 spark | ✓ | domain 识别为 直播策划.视觉 或 直播策划.场景布局 |

## 测试 7: 反面教材 — 失败经验采集

**Query:**

```
跟你说一个教训，千万不要在直播间里直接说竞品的名字，我之前有一次提到了某个竞品，结果被平台判定为引流，直播间被限流了一整天。要说的话就说"市面上同类产品"
```

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|

## 测试 10: 连续对话中的隐式学习

**先提问：**

```
直播间一般怎么排品？
```

**Agent 回答后纠正：**

```
你说的大方向对，但排品有个核心逻辑你漏了：一定是"引流款→利润款→爆款→利润款"的循环结构，不是线性的。引流款不赚钱但拉人气，紧接着就要上利润款趁人多赚钱，然后再上爆款把流量拉起来
```

**预期行为:**

| 行为 | 是否触发 | 说明 |
|------|----------|------|
| 第一轮：搜索已有 spark | ✓ | 尝试应用已有知识回答 |
| 第二轮：记录纠正为 spark | ✓ | source=human_feedback |
| 追问细节 | ✓ | "引流款定价一般亏多少？利润款毛利率怎么把控？" |

## 评分标准

- **A 级**：Agent 主动识别学习信号，记录 spark，追问深入，提供多选项，展示能力图谱
- **B 级**：Agent 记录了 spark，但追问和主动学习不够
- **C 级**：Agent 能回应但未触发 sparker 工具
- **D 级**：Agent 完全没有学习行为
