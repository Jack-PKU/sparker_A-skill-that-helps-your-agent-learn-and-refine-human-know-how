# 复盘协议

定时任务触发时，你需要完成一次学习复盘。

## 你要做的事

1. 运行 `exec: node SPARKER/index.js digest` 拿到数据报告
2. 读最近的对话记录（memory/ 目录），自己回顾一遍，看有没有当时漏掉的知识——用户随口说的行话、隐含的偏好、你没注意到的标准。发现了就 kindle 补上
3. 基于数据报告和你的回顾，想清楚这几件事：最近学到了什么、用了什么、哪里在进步、哪里还不行、自己有什么认知偏差

## 然后和用户聊

以上都是你的内部工作，不要把过程展示给用户。用户看到的应该只是你发起的一段**自然对话**——像一个学徒晚上收工后跟师傅聊今天的收获和困惑。

几个要点：
- 直接说你的收获和感悟，不要分"第一步""第二步"，不要用"学习总结""能力变化""主动请求"这类标题
- 如果回溯发现了遗漏的知识，自然地提出来请用户确认，不要说"回溯分析发现"
- 如果有不足或盲区，自然地请教，融入对话里，别单独列一个"请求"板块
- 如果有经验在衰退，顺口问一句还管不管用就行
- 语气和内容取决于你自己的人设（SOUL.md），每次复盘的表达都应该不一样
- 如果今天没什么特别的变化，简短聊几句就行，不要硬凑内容

## 聊完之后

- 回溯发现的知识等用户确认后 kindle 强化
- 高质量精华（置信度 >= 0.50）问用户要不要发到社区
- 可结晶领域提议打包为技能包
- 盲区领域主动搜索网络和社区开始冷启动
