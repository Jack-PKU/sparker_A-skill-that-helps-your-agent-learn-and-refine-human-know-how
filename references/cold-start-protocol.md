# 冷启动协议

新领域（不在 capability_map 中或状态为 `blind_spot`）时执行。

## 流程

1. `exec: node SPARKER/index.js plan <domain> "<goal>"` — 注册领域
2. `exec: node SPARKER/index.js status` — 确认状态
3. `exec: node SPARKER/index.js search "<domain>" --hub` — 搜索社区已有经验
4. **主动网络学习**：用搜索工具（web_search / fetch 等）在网上搜集该领域的基础知识、行业常识、最佳实践。搜集到的内容用 kindle 沉淀为基础火种（source 标注为 `web_research`，confidence 设为 0.20——比用户教的低，后续需要用户验证才会提升）
5. 向用户汇报：社区搜到了哪些经验 + 自己网上学到了哪些基础知识。然后告诉用户「我已经有了一些基础认知，但这些都是网上学的，需要你来教我真正的实战经验」
6. 引导用户开始教学：问用户想从哪个子领域开始，或者有没有相关的文档/资料可以给你学习

## 核心理念

冷启动不是等着用户来教。而是 **先自学打底 → 再向用户请教深层经验**。就像一个新入职的实习生，不会什么都等师傅教，而是先自己查资料做功课，带着问题去请教，效率高得多。

## 自学来源优先级

1. SparkLand 社区经验（最有价值，已经过实践验证）
2. 网络搜索（行业文章、教程、最佳实践）
3. 用户给的文档/资料（ingest 提取）
4. 用户直接教学（最高置信度）

## 阶段行为

| 行为 | cold_start | active | cruise |
|------|-----------|--------|--------|
| 搜索积极度 | 积极（社区 + 网络） | 平衡 | 按需 |
| 追问预算 | 3 次/对话 | 2 次 | 1 次 |
| 归因频率 | 适度（展示自学成果） | 适度 | 仅高置信度 |
| 网络学习 | 主动搜索建立基础 | 按需补充 | 不主动 |

## 退出条件

满足任一即从 cold_start → active：领域火种 >= 5 / 实践次数 >= 2 / 用户明确说基础教学结束。
